IoT Honeypot & Threat Analysis

A virtualised IoT honeypot environment built to capture and analyse real-world attacker behaviour. 552 authentication attempts were logged and analysed using a custom Python tool, revealing credential patterns, attack origins, and attacker methodology.

Total Attempts 552
Failure Rate 98.4%
Duration 7 Days
Platform VirtualBox / Kali

Project Overview

This project was developed as a Final Year Project to investigate how IoT devices are targeted by automated credential attacks. A Cowrie SSH/Telnet honeypot was deployed in a VirtualBox virtualised environment to safely capture attacker interactions without exposing real systems.

The system ran continuously, logging all inbound connection attempts, credential pairs used, commands executed, and session metadata. A custom Python analysis tool was written to process the raw logs and generate statistical reports.

[ Screenshot: System Architecture Diagram ]

Technical Approach

  • Deployed Cowrie honeypot on Ubuntu VM via VirtualBox with iptables redirect on port 22
  • Configured fake IoT environment to attract automated scanning traffic
  • Captured 552 authentication attempts over a 7-day capture window
  • Built enhanced_credential_analyzer.py to parse logs and extract statistical insights
  • Analysed top credential pairs, attack timing patterns, and geographic source data
  • Produced formal threat analysis report with remediation recommendations

Key Finding

98.4% of all 552 authentication attempts failed, with "admin/admin" and "root/root" accounting for over 40% of credential attempts — demonstrating the scale of automated IoT credential scanning.

[ Screenshot: Attack frequency chart / Python analysis output ]

Results & Analysis

The capture period produced a dataset of 552 individual authentication attempts from 87 unique source IPs. Attack activity was consistent across all hours, suggesting fully automated scanning bots rather than manual attackers.

  • Top credential pair: admin / admin (used in 22% of attempts)
  • Attack traffic peaked between 02:00 – 06:00 UTC
  • 87 unique source IPs identified across 14 countries
  • All failed attempts confirm credential stuffing as primary attack vector

Code Highlight

Custom Python credential analyser — processes Cowrie logs and outputs ranked credential data:

# enhanced_credential_analyzer.py (excerpt)
def analyse_credentials(log_path):
    attempts = parse_cowrie_logs(log_path)
    credential_counts = Counter(attempts)
    return credential_counts.most_common(10)
